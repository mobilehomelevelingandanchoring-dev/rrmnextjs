## BERT/NLP OPTIMIZATION STRATEGY

### Question-Answer Coverage
Every page must answer:
1. **What?** "What is [service/problem]?"
2. **Why?** "Why do I need this?" "Why is this important?"
3. **How?** "How is this done safely?"
4. **When?** "When should I get this service?"
5. **Where?** "Where do you serve?" (For local intent)
6. **Cost?** "How much does this cost?" (Implied answer in CTA)

### Semantic Keywords (Not Forced, Natural Integration)
- **Service variations:** Cleaning, washing, removal, treatment, care, maintenance
- **Quality markers:** Professional, safe, eco-friendly, certified, expert
- **Problem outcomes:** Restore, prevent, protect, extend longevity
- **Climate context:** UK weather, moisture, humidity, seasonal maintenance
- **User intent:** DIY vs professional, costs, time, results

### Latent Semantic Indexing (LSI) Terms
- **For Moss Removal:** Spores, regrowth, biocide, soft wash, moisture, freeze damage, tile displacement

## TECHNICAL REQUIREMENTS

### URL Structure
- `/services/[service-slug]` - Service pages
- `/locations/[location-slug]` - Location pages
- `/problems/[problem-slug]` - Problem entity pages
- `/surfaces/[surface-slug]` - Surface entity pages

### Routing (React Router)
```jsx
<Route path="/services/:serviceSlug" element={<ServicePage />} />
<Route path="/locations/:locationSlug" element={<LocationPage />} />
<Route path="/problems/:problemSlug" element={<ProblemPage />} />
<Route path="/surfaces/:surfaceSlug" element={<SurfacePage />} />
```

### Data Structure
```typescript
interface Service {
  slug: string;
  title: string;
  description: string;
  longDescription: string;
  problems: string[]; // Problem entity slugs
  surfaces: string[]; // Surface entity slugs
  locations: string[]; // Location slugs
  relatedServices: string[]; // Service slugs
  faqs: { q: string; a: string }[];
}

interface Location {
  slug: string;
  name: string;
  description: string;
  services: string[]; // All services available
  localContext: string; // Climate, weather, common issues
  nearbyLocations: string[]; // Nearby towns
}

interface Problem {
  slug: string;
  title: string;
  description: string;
  solutions: string[]; // Service slugs that solve it
  affectedSurfaces: string[]; // Surface slugs
}

interface Surface {
  slug: string;
  name: string;
  description: string;
  services: string[]; // Services for this surface
  problems: string[]; // Problems common to this surface
}
```
